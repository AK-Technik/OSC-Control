# Behringer X32 — OSC Protocol Notes

## Transport

- **Protocol:** OSC over UDP
- **Remote port:** `10023` (fixed, not configurable on the desk)
- **Local/source port:** Must be **fixed** — see Source Port Gotcha below

---

## /xremote subscription

The X32 only sends feedback (pushed state updates) to a client that has an active subscription. To subscribe:

```
OSC path: /xremote
Arguments: (none)
```

Send this packet to the X32 every **< 10 seconds**. The subscription lapses exactly 10 seconds after the last `/xremote` received. A safe keepalive interval is **8 seconds**.

Without an active subscription, the X32 will still respond to queries (get-style `/node` requests) but will not push fader/mute changes triggered by the physical desk or other clients.

### Source port gotcha

The X32 sends feedback to the **source IP:port** of the most recently received `/xremote` packet. If your client uses an ephemeral (OS-assigned) UDP port, the source port changes between restarts and the X32 feedback goes to the wrong port.

**Fix:** Bind the client socket to a fixed local port before sending `/xremote`. The showcontrol integration uses `source_port` in the transport block for this. Recommended value: `10024`.

## Querying state (get)

Send an OSC message with the path only, no arguments:

```
/ch/01/mix/fader   → X32 replies: /ch/01/mix/fader <float 0.0–1.0>
/ch/01/mix/on      → X32 replies: /ch/01/mix/on <int 0 or 1>
```

You can also use `/node` for bulk queries:

```
/node   "ch/01/mix"   → returns a formatted string of all sub-parameters
```

## Fader values

Faders are `float` in the range `0.0 – 1.0`. This is **not linear dB**; it's the X32's internal taper.

Approximate landmarks:

| Float value | Approximate dB |
|---|---|
| 0.0 | −∞ (−inf) |
| 0.25 | −50 dBFS |
| 0.5 | −30 dBFS |
| 0.75 | 0 dBFS (unity) |
| 1.0 | +10 dBFS |

Use `value_type: float` in profiles.

## Mute semantics (important — inverted)

The X32 `/mix/on` parameter means **"is the channel ON (active/unmuted)"**, not "is it muted":

| OSC value | Meaning |
|---|---|
| `1` | Channel is **ON** → unmuted, signal passes |
| `0` | Channel is **muted** → signal blocked |

This is counterintuitive for a "mute" button. In showcontrol profiles, set `bool_inverted: true` on mute entities so that:

- HA switch `on` → sends `0` → channel muted
- HA switch `off` → sends `1` → channel unmuted

Mute groups (`/-stat/mute/{n}`) use the same convention: `1` = mute group **active** (muting its members).

## Meter subscriptions

To receive metering data (RMS/peak levels), send:

```
OSC path: /meters
Arguments: <string> "/meters/1"   (or /meters/2 .. /meters/15 for different meter pages)
           <int>    0              (refresh rate hint — 0 = fast)
```

Meter data arrives as blobs. Decoding requires unpacking 32-bit IEEE floats from the blob. This is not used by the showcontrol integration's standard entity platforms.

---

## OSC Bundle messages

The X32 supports OSC bundles (multiple messages in one UDP packet with a common timestamp). This can reduce round-trips for multi-parameter snapshots. The showcontrol integration sends individual messages by default. Bundle support may be added as an optimization.

---

## Scene recall

Scenes are zero-indexed (scene 1 on the desk = index 0 in OSC):

```
OSC path: /-action/goscene
Arguments: <int> scene_index   (0–99)
```

There is no feedback when a scene completes loading. If you need confirmation, poll the scene name:

```
/-show/prepos/current   → <int> current scene index
```

## DCA path reference

```
/dca/{1-8}/fader    float   DCA group fader (0.0–1.0)
/dca/{1-8}/on       int     DCA on/off (1=on/unmuted, 0=muted) — same inverted convention
/dca/{1-8}/name     string  DCA name label
```

---

## Common path reference

```
/ch/{01-32}/mix/fader     float   Channel fader
/ch/{01-32}/mix/on        int     Channel on (1=on)
/ch/{01-32}/mix/pan       float   Pan (0.0=left, 0.5=center, 1.0=right)
/auxin/{01-08}/mix/fader  float   Aux in fader
/bus/{01-16}/mix/fader    float   Mix bus fader
/mtx/{01-06}/mix/fader    float   Matrix fader
/main/st/mix/fader        float   Main LR fader
/main/m/mix/fader         float   Main mono/center fader
/dca/{1-8}/fader          float   DCA fader
/-stat/mute/{1-6}         int     Mute group active (1=muting)
/-action/goscene          int     Recall scene by index
/-action/tapbeat          int     Tap tempo (send 1)
/xremote                  (none)  Subscribe to feedback
```

## Tested firmware

These notes apply to X32 firmware **4.06**. Behringer has changed OSC behavior in past firmware updates; verify paths on significantly different versions.
